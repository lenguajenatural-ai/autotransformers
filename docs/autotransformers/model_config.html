 10
 11    With this we determine every choice related to the model
 12    such as the original name, the name to save the model with, the hyperparameter space, and a long etc.
 13
 14    Parameters
 15    ----------
 16    name: str
 17        Name of the model, either in the HF hub or a path to the local directory where it is stored.
 18    save_name: str
 19        Alias for the model, used for saving it.
 20    hp_space
 21        The hyperparameter space for hyperparameter search with optuna. Must be a function receiving a trial and returning a dictionary with the corresponding suggest_categorical and float fields.
 22    dropout_vals: List
 23        Dropout values to try.
 24    custom_config_class: transformers.PretrainedConfig
 25        Custom configuration for a model. Useful for training ensembles of transformers.
 26    custom_model_class: transformers.PreTrainedModel
 27        Custom model. None by default. Only used for ensemble models and other strange creatures of Nature.
 28    partial_custom_tok_func_call: Any
 29        Partial call for a tokenization function, with all necessary parameters passed to it.
 30    max_length_summary: int
 31        Max length of the summaries. Useful for summarization datasets.
 32    min_length_summary : int
 33        Min length of the summaries. Useful for summarization datasets.
 34    no_repeat_ngram_size: int
 35        Number of n-grams to don't repeat when doing summarization.
 36    early_stopping_summarization: bool
 37        Whether to have early stopping when doing summarization tasks.
 38    length_penalty: float
 39        Length penalty for summarization tasks.
 40    num_beams: int
 41        Number of beams in beam search for summarization tasks.
 42    dropout_field_name: str
 43        Name for the dropout field in the pooler layer.
 44    n_trials : int
 45        Number of trials (trainings) to carry out with this model.
 46    random_init_trials: int
 47        Argument for optuna sampler, to control number of initial trials to run randomly.
 48    trainer_cls_summarization: Any
 49        Class for the trainer. Useful when it is desired to override the default trainer cls for summarization.
 50    model_cls_summarization: Any
 51        Class for the trainer. Useful when it is desired to override the default trainer cls for summarization.
 52    custom_tokenization_func: Any
 53        Custom tokenization function for processing texts. When the user does not want to use the default tokenization function for the task at hand, one can create a custom tokenization function. This function must receive samples from a dataset, a tokenizer and a dataset config.
 54    only_test: bool
 55        Whether to only test, not train (for already trained models).
 56    test_batch_size: int
 57        Batch size for test; only used when doing only testing.
 58    overwrite_training_args: Dict
 59        Arguments to overwrite the default arguments for the trainer, for example to change the optimizer for this concrete model.
 60    save_dir: str
 61        The directory to save the trained model.
 62    push_to_hub: bool
 63        Whether to push the best model to the hub.
 64    additional_params_tokenizer: Dict
 65        Additional arguments to pass to the tokenizer.
 66    resume_from_checkpoint: bool
 67        Whether to resume from checkpoint to continue training.
 68    config_problem_type: str
 69        The type of the problem, for loss fct.

 75        Dictionary with custom parameters for loading AutoConfig.
 76    generation_params: Dict
 77        Parameters for generative tasks, for the generate call.
 78    hf_hub_username: str
 79        Username in HF Hub, to push models to hub.
 80    custom_results_getter: Any
 81        Custom class to get test results after training.
 82    quantization_config: BitsAndBytesConfig
 83        Configuration for quantizing models.
 84    custom_params_model: Dict
 85        Dictionary with custom parameters for initializing the model.
 86    model_init_wrap_cls: Any
 87        Wrap class with a `__call__` method for the `model_init` function, useful for example for preparing quantized models, etc.
 88    peft_config: Any
 89        Configuration class from Peft. For example, `LoraConfig`.
 90    func_modify_tokenizer: Any
